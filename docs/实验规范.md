# 实验规范

本文档用于约束 AP-SRR-PSO 项目的实验记录方式，避免后期论文写作时出现数据来源混乱、算法名称不一致、统计结果无法复现等问题。

## 一、统一命名

| 类型 | 名称 |
|---|---|
| 主方法 | AP-SRR-PSO |
| 旧方法 | ARPSO-SRR |
| 标准粒子群 | PSO |
| 不再使用 | ARPSO-v4 |

旧结果中如果存在 `ARPSO-v4`，在新仓库中统一映射为 `ARPSO-SRR`。

## 二、随机种子

所有独立运行必须记录随机种子。

建议规则：

```text
seed = base_seed + function_id * 1000 + algorithm_id * 100 + run_id
```

这样不同函数、不同算法、不同 run 之间不会混淆。

## 三、结果 CSV 字段

正式实验结果至少包含：

```text
benchmark
function
algorithm
dimension
run
seed
best_fitness
function_evaluations
runtime_seconds
restart_count
metadata
```

消融实验额外建议包含：

```text
variant
ips_enabled
arp_enabled
rca_enabled
operator_usage
operator_success
```

## 四、主实验设置

CEC2017 30D 建议：

```text
population_size = 50
max_fes = 300000
runs = 30
```

CEC2017 50D 建议：

```text
population_size = 50
max_fes = 500000
runs = 30
```

## 五、统计检验

最终论文至少需要：

1. Mean / Std；
2. Wilcoxon signed-rank test；
3. Friedman average rank；
4. Holm post-hoc correction；
5. Win/Tie/Loss；
6. 收敛曲线；
7. 运行时间分析；
8. 消融实验。

## 六、图表规范

论文图片优先使用：

- PDF；
- SVG；
- TikZ。

不要使用低分辨率截图。

## 七、旧成果迁移原则

`legacy/` 目录只保留旧项目证据，不直接修改。新 SCI 代码应放在当前标准目录中。

建议迁移：

```text
legacy/code/
legacy/results/
legacy/paper/
legacy/docs/
```

不建议迁移：

```text
临时压缩包
LaTeX 编译缓存
重复图片
无来源说明的中间文件
```
