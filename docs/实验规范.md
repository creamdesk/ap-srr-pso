# 实验规范

本文档用于约束 AP-SRR-PSO 项目的实验记录方式，避免后期论文写作时出现数据来源混乱、算法名称不一致、统计结果无法复现等问题。

论文主线以 `docs/PAPER_STRATEGY.md` 为准。所有实验脚本、分析脚本、图表和表格都应服务于：

```text
PSO premature convergence
-> ineffective particle identification
-> search resource waste
-> adaptive search resource reallocation
-> restart operator portfolio
-> credit assignment
-> more robust optimization behavior
```

## 一、统一命名

| 类型 | 名称 |
|---|---|
| 主方法 | AP-SRR-PSO |
| 旧方法 | ARPSO-SRR |
| 标准粒子群 | PSO |
| 不再使用 | ARPSO-v4 |

旧结果中如果存在 `ARPSO-v4`，在新仓库中统一映射为 `ARPSO-SRR`。

## 二、随机种子

所有独立运行必须记录随机种子。

建议规则：

```text
seed = base_seed + function_id * 1000 + algorithm_id * 100 + run_id
```

这样不同函数、不同算法、不同 run 之间不会混淆。

## 三、结果 CSV 字段

正式实验结果至少包含：

```text
benchmark
function
algorithm
dimension
run
seed
best_fitness
function_evaluations
runtime_seconds
restart_count
metadata
```

消融实验额外建议包含：

```text
variant
ips_enabled
arp_enabled
rca_enabled
operator_usage
operator_success
```

## 四、主实验设置

实验等级必须区分：

| 等级 | 目的 | 是否可作为正式论文结果 |
|---|---|---|
| smoke test | 环境和输出自检 | 否 |
| mini validation | CEC adapter、CSV、summary 流水线验证 | 否 |
| 30D probe | 30D 稳定性和耗时估计 | 否 |
| 30D pilot | 小规模多函数验证 | 否 |
| formal 30D | 正式统计实验 | 是 |

不要把 2 runs 或 5 runs 的工程验证写成统计显著性结论。

CEC2017 30D 建议：

```text
population_size = 50
max_fes = 300000
runs = 30
```

CEC2017 50D 建议：

```text
population_size = 50
max_fes = 500000
runs = 30
```

## 五、统计检验

最终论文至少需要：

1. Mean / Std；
2. Wilcoxon signed-rank test；
3. Friedman average rank；
4. Holm post-hoc correction；
5. Win/Tie/Loss；
6. 收敛曲线；
7. 运行时间分析；
8. 消融实验。

## 六、图表规范

论文图片优先使用：

- PDF；
- SVG；
- TikZ。

不要使用低分辨率截图。

禁止只输出 PNG。正式论文图至少输出 PDF 和 SVG，并同步到 `paper/figures/`。

## 六点五、禁止表述

论文和文档中避免：

- state-of-the-art；
- universally best；
- always outperforms；
- no computational overhead；
- 用 mini/probe/pilot 数据证明算法优越；
- 隐藏失败函数或只展示好看的曲线。

允许使用：

- competitive performance；
- improved robustness on selected complex functions；
- effective search resource reallocation；
- interpretable restart behavior；
- acceptable additional computational cost。

## 七、旧成果迁移原则

`legacy/` 目录只保留旧项目证据，不直接修改。新 SCI 代码应放在当前标准目录中。

建议迁移：

```text
legacy/code/
legacy/results/
legacy/paper/
legacy/docs/
```

不建议迁移：

```text
临时压缩包
LaTeX 编译缓存
重复图片
无来源说明的中间文件
```
